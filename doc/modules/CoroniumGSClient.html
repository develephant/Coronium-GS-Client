cancelGame ()
Cancel a Game in a waiting state. Useful when no games found. You will revieve GameCancel when the game has been canceled.

Usage:

    gs:cancelGame()
connect (connection_table)
Connect to a listening game server instance.

Parameters:

  • connection_table See connection_table
createGame (players_max, game_criteria)
Create a new game for other players to connect to. You can only use this method after the ClientConnect client-side event. Only call createGame OR joinGame once in each client, not both.

Parameters:

  • players_max int The maximal amount players needed to start this game.
  • game_criteria table Special criteria for other players to find the game with. This is useful for private, friend to friend, or matchmaking games. The game_criteria table supports a 'tag' table key that will be matched up with the joinGame method. This creates a way to direct clients to specific rooms. See also joinGame .

Usage:

     function onClientConnect( event )
   --== Create a 2 player game for others to join.
   gs:createGame( 2 )
   --== Create a 2 player game with a custom 'tag' for the other players to join with.
   gs:createGame( 2, { tag = 'custom_str' } )
 end

getPlayersInfo ()
Get the game players info data table. This method can only be called after the GameCreate or GameJoin client-side event.

Returns:

    table A table of game players meta data.

Usage:

     function onGameJoin( event )
  local info_table = gs:getPlayersInfo()
  for key, value in pairs( info_table ) do
    p( value )
  end
 end

joinGame (players_max, game_criteria)
Join a game that was created with the createGame method. You can only use this method after the ClientConnect client-side event. Only call createGame OR joinGame once in each client, but not both.

Parameters:

  • players_max int Join a game with players_max players.
  • game_criteria table Special criteria to find game by. This table supports a 'tag' key which you can use to find specifically created games. See also createGame .

Usage:

     function onClientConnect( event )
 --== Find a 2 player game to join.
  gs:joinGame( 2 )
 --== Find a 2 player game with a custom 'tag' to join.
  gs:joinGame( 2, { tag = 'custom_str' } )
 end

Tables

connection_table
The connection table used for the connect method. A key can be set on the server to help limit unwanted connections. You must pass a matching key in the key parameter to successfully pair the client to the server. By enabling the ping parameter, you keep the client alive and connected. This basically disables any client timeout settings on the server-side. The handle parameter can be used as a username, player name, or any other identifier. Additional starting data can be sent as a simple table in the data parameter. You can retrieve this data with the client:getPlayerData() method. See Client.

Fields:

  • host string The host address.
  • port int The host port. (default 7173)
  • handle string String based identifier for the client. (default server assigned)
  • data table A simple starting data table for the client. (default nil)
  • key string The server pairing key. (default 'abc')
  • ping bool The ping flag. (default false)

Usage:

     local conn_tbl =
 {
  host = 'ping.coronium.gs',
  port = '7173',
  handle = 'Chris',
  data = { color = "blue", fun = "Yes!" },
  key = 'abc',
  ping = true
 }
gs:connect( conn_tbl )

Game Events

You can listen for the following Game events.
GameCancel
The game has been canceled. State reset.
  • GameCancel

Usage:

    gs.events:on( "GameCancel", function( event )
	print( "game canceled" )
end )
GameCreate
A game has been created. This event will only be received by the client/player calling it.
  • GameCreate

Usage:

    gs.events:on( "GameCreate", function( event )
	print( event.data )
end )
GameData
Game data has been received.
  • GameData

Usage:

    gs.events:on( "GameData", function( event )
	p( event.data.game_data )
end )
GameDone
The game is finished. See the server-side call publishGameDone as well.
  • GameDone

Usage:

    gs.events:on( "GameDone", function( event )
	print( event.data )
end )
GameJoin
A player has joined the game.
  • GameJoin

Usage:

    gs.events:on( "GameJoin", function( event )
	print( event.data )
end )
GameLeave
A player has left the game.
  • GameLeave

Usage:

    gs.events:on( "GameLeave", function( event )
	print( event.data )
end )
GameStart
The game has started. All players present.
  • GameStart

Usage:

    gs.events:on( "GameStart", function( event )
	print( event.data )
end )
